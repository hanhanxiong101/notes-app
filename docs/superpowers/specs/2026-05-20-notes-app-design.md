# 笔记应用设计文档

## 概述

一个基于 React + Vite 构建的个人笔记网页应用，支持搜索、标签分类、图片上传，数据存于浏览器 localStorage。

## 视觉风格

温暖纸张风：暖色调（米色背景 #f5f0e8，棕色点缀 #8b7355），仿纸质笔记本的舒适氛围。

## 页面流程

```
列表页（首页） → 点击笔记 → 详情页
列表页（首页） → 点击"新建" → 编辑页
详情页 → 点击"编辑" → 编辑页
编辑页 → 保存 → 返回列表页
详情页 → 删除 → 确认框 → 返回列表页
```

## 数据模型

```js
{
  id: "1694236800000",
  title: "今天的想法",
  content: "今天学到了...",
  tags: ["学习", "React"],
  images: ["data:image/png;base64,..."],
  createdAt: "2026-05-20",
  updatedAt: "2026-05-20"
}
```

## 组件树

```
App
├── Header（搜索框 + 新建按钮）
├── TagFilter（标签筛选栏）
├── Router
│   ├── NoteList（列表页）
│   │   └── NoteCard（卡片：标题、摘要、标签、日期）
│   ├── NoteDetail（详情页：内容、标签、时间、操作按钮）
│   └── NoteEditor（编辑页：标题、内容、标签、图片上传）
└── EmptyState（空状态提示）
```

共 8 个组件。

## 边界情况

| 场景 | 处理 |
|------|------|
| 没有笔记 | 空状态提示 + 引导 |
| 搜索无结果 | "没有找到匹配的笔记" |
| 标题为空 | 自动用"（无标题）" |
| 内容为空 | 允许保存，卡片显示"（空内容）" |
| 删除确认 | 弹出确认框 |
| 编辑取消 | 返回上一页，不保存 |
| 图片过大 | 提示限制 500KB |

## 不做的事

- 用户登录、云端同步
- Markdown 编辑器
- 笔记恢复/回收站

## 技术栈

- React 19 + Vite
- React Router（页面路由）
- 纯 CSS（温暖纸张风配色）
- localStorage 存储
